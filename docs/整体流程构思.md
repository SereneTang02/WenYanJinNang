## 一、整体流程（学生视角）

1. **打开小程序（首页 =「实词首页」）**
   - 如果是第一次进入：
     - 让用户选年级（七上/七下/八上/八下/九上/九下），写入本地 `userGrade`。
   - 展示 4 个入口：
     1. 「刷实词卡片」（基础功能，免费）
     2. 「今日打卡」（X 个旧词 + Y 个新词）
     3. 「小古文记词」
     4. 「班级/PK」+「AI讲解」（高级功能）
2. **每次进入首页时：**
   - 根据今天日期，检查是否已经生成“今日任务词表”：
     - 没有：按你的策略抽：`X个旧词 + Y个新词`，存到 `dailyWords_YYYYMMDD`。
     - 有：直接读取。
   - 读取本地“错题统计”与“打卡记录”，更新首页展示：
     - 今日任务完成进度（已完成题数 / 总题数）
     - 已连续打卡天数（为 30 天计划做准备）
     - 错题数（可展示“炼丹炉/错词炉”入口）

## 二、模块 1：实词卡片（免费基础）

> 图上：✅「实词卡片——目标常词 免费；可以做义项刷题；可以分年级」

### 1.1 数据结构（逻辑上）

在不动数据集的前提下，运行时维护一个“学习状态表”`userWordProgress`（存本地，或者将来存在云端）：

```ts
userWordProgress = {
  [wordId: string]: {
    correct: number,   // 做对次数
    wrong: number,     // 做错次数
    lastReviewAt: Date,
    nextReviewAt: Date, // 为以后排复习用
  }
}
```

### 1.2 卡片列表页（word-list）

- 载入 `wordsData`，按 `userGrade` 默认过滤出本年级重点词。
- 每个词卡上显示：
  - 词 + 拼音 + 年级 + 是否“高频”等标签。
  - **颜色由错题情况决定：**
    - `wrong == 0` → 白色
    - `wrong > 0 & wrong - correct < 3` → 浅黄
    - `wrong - correct ≥ 3` → 橙/红
- 点击卡片 → 跳转到详情页。

### 1.3 卡片详情页（word-detail）

- 显示：所有义项 + 例句 + 考点说明。
- 最下面两个按钮：
  1. 「用这个词出几道题练一练」→ 跳 practice 页，参数 `wordId`。
  2. 「加入今日任务」→ 把此 `wordId` 加入今天的 `dailyWords`（供打卡使用）。

## 三、模块 2：每日打卡 + 错题库（颜色加深逻辑）

> 图上：黑字“每日打卡（x个旧词，y个新词）”，“错题库：可以用颜色来呈现题错次数，错得越多颜色越深，需要答对若干次恢复。”

### 2.1 生成今日任务

1. 打开首页时：
   - 查 `localStorage['dailyTaskDate']` 是否等于今天：
     - 若不同：重新生成今日任务：
       - 从 `userWordProgress` 里挑“到期复习”的旧词（`nextReviewAt <= today`），取 X 个。
       - 再从 `wordsData` 中随机挑重要但没学过的词，取 Y 个。
       - 合并成 `todayWordIds`，写入 `dailyTaskDate`、`dailyWordIds`、`dailyFinishedCount=0`。
2. 首页“今日打卡”按钮 → practice 页，参数带 `wordIds = todayWordIds`。

### 2.2 练习页面（practice）

- 接收参数 `wordIds`（可能来自今日任务，也可能只一个词）。

- 出题逻辑：

  - 对每个 `wordId` 的义项+例句生成 1～2 道选择题。

  - 题目结构：

    ```ts
    {
      id,
      wordId,
      questionType: 'choice',
      sentence,          // 例句
      options: [ { label, isCorrect }, ... ]
    }
    ```

- 用户选择后：

  - 立刻判对错，展示“答对/答错 + 正确解析”。
  - 更新 `userWordProgress[wordId].correct/wrong`。
  - 同时更新“颜色深浅”。

### 2.3 颜色逻辑 & 错题库页面

- 颜色等级可以简单用函数实现：

```js
const diff = progress.wrong - progress.correct;
if (diff <= 0) color = 'white';
else if (diff <= 2) color = 'yellow';
else color = 'orange';
```

- “错题库”单独一个页面：
  - 从 `userWordProgress` 里筛选 `wrong > 0` 的词，按 diff 倒序显示。
  - 点击直接开始“专项纠错练习”。

## 四、模块 3：班级房间 + PK

> 图上红色：✅「班级房间：老师建房间，学生用邀请码加入，PK 闯关量表 + 榜单，PK 结果老师获得数据报告，学生获得错题分析报告。（老师 id 赠 1 次免费试用）」

这里逻辑分“老师端”和“学生端”，可以先做最简单版：
 数据暂时放云开发数据库 / 你自己的后端。

### 4.1 老师端：创建班级 & 房间

1. 老师登录（用微信 openid + 手动选择“我是老师”）。
2. 在「班级」页点击“新建班级”：
   - 输入班级名、年级。
   - 系统生成一个 `joinCode`，写入 `classes` 表。
3. 在班级内部创建“PK 任务”：
   - 选择词库范围（按年级、难度、专题）。
   - 设置题目数量/时间。
   - 生成 `pkSessionId`，状态为 `pending`。

### 4.2 学生端：加入班级 & 参加 PK

1. 学生在「班级」页输入老师提供的邀请码 `joinCode`：
   - 服务器校验，写入 `class_members`。
2. 老师发起 PK 时：
   - 后端把该 `pkSession` 状态改为 `running`，通知班级所有学生（轮询 / 云推送）。
3. 学生点“立即参加 PK”：
   - practice 页以 `mode = "pk"` + `pkSessionId` 打开。
   - 出题逻辑与普通练习类似，只是：
     - 所有题目统一来自同一个 `pkSession` 的题目列表。

### 4.3 结果与榜单

- 学生端：
  - PK 结束 → 显示个人得分、用时、错题列表。
- 老师端：
  - 在班级页面查看某个 `pkSession` 的统计：
    - 每个学生得分、用时；
    - 每个词的错误率（用来指导讲解）。

> 逻辑上就是：
>  **pkSession** 定义题目集合 → 学生作答生成 **pkRecords** → 汇总出班级报告。

## 五、模块 4：AI 讲解（限次数 + token 计费）

> 图上蓝字：✅「AI 讲解 —— 调用大模型言模型；每月可免费进行 3 次对话（按 token 来计费）」

### 5.1 配额逻辑

为每个用户维护一个 `aiUsage` 状态（可存后端，也可本地+云同步）：

```ts
aiUsage = {
  month: '2025-11',     // 当前计费月
  freeCountUsed: 2,     // 当月已使用免费对话次数
  tokenUsed: 12345      // 总 token 数（用于统计成本）
}
```

每次调用 AI 前：

1. 检查当前年月是否与 `aiUsage.month` 一致：
   - 不一致：重置为当前月，`freeCountUsed = 0`。
2. 如果 `freeCountUsed < 3`：
   - 允许本次对话，`freeCountUsed++`。
3. 如果 `freeCountUsed >= 3`：
   - 弹出提示：“本月免费次数已用完，开通会员继续使用”。

### 5.2 调用内容

在任意实词详情页或练习解析处，有「问 AI 小先生」按钮：

- 传入参数：
  - 当前词 / 当前题目的句子 / 用户具体问题。
- 后端封装成 prompt 调用大模型：
  - 要求输出：
    - 更白话的解释；
    - 2～3 个新例句；
    - 考试易错点提醒。
- 前端展示 AI 返回的内容，并记录 token 消耗。

## 六、模块 5：会员 / 机构版收费（同建契约）

> 图上红字：✅「同建契约 或 团卡号做混合收费：教师版/机构版（网页封装，学生可直接手机端体验）」

这里的“执行逻辑”可以先定义清楚**权限差异**，以后接支付接口即可。

### 6.1 个人会员（学生/家长）

- 免费版可以使用：
  - 刷实词卡片、每日打卡、基础练习；
  - 班级 PK 参与（由老师开通）。
- 会员版新增权益：
  - 解锁全部“30 天打卡计划”；
  - AI 讲解不限次数或次数大幅增加；
  - 查看详细学习报告（掌握度曲线、一词多义掌握情况等）。

逻辑上，只要有一个 `userRole` 和 `vipExpireAt` 即可：

```ts
user = {
  ...,
  vipExpireAt: Date | null
}

function isVip(user) {
  return user.vipExpireAt && user.vipExpireAt > today;
}
```

### 6.2 教师版 / 机构版

- 老师付费后：
  - 可以创建多个班级、多个 PK 活动；
  - 解锁“班级报告导出”、“错题册 PDF”等功能；
  - 每个老师账户绑定若干学生免费使用 AI 讲解 N 次。
- 机构版：其实是开通多个教师账号 + 统一账单。

> 对 Claude Code 来说，只需要先预留 `isVip`、`isTeacher`、`isOrg` 这些字段与判断逻辑，真正支付接入可以后置。

## 七、模块 6：组词成文（小古文记词）

> 图下粉色：⭐「只是词汇？→ 组词成文」
>  ① 用户选一组要记的词和义项
>  ② 系统初步生成一篇小古文
>  ③ 高亮目标词，点词弹出释义
>  ④ 剧情模式，同组之中的句子和对话，制作脚本卡

### 7.1 入口 & 选词

1. 在首页或实词列表页有入口「小古文记词」。
2. 进入后步骤：
   - Step1：让用户勾选 4～8 个要记的词（可以按专题推荐，如“学习成长”“山水游记”）。
   - Step2：可选：选择“难度、文风”（简单 / 正经 / 对话体）。

### 7.2 生成小古文

- 前端把选好的词 ID + 义项信息发给后端。
- 后端调用大模型，提示：“请用这些词写一段 120～180 字的小古文，尽量每个词出现 1～2 次，并保证意义自然。”
- 返回文本后：
  - 前端在文本中“查找并高亮这些词”，将它们做成可点击组件。

### 7.3 展示与练习

- 阅读模式：
  - 高亮词可点击，弹出该词释义+例句+所属义项。
- 练习模式：
  - 第一轮：完整阅读 + 点击查看释义；
  - 第二轮：将这些词替换为空格或“□”，做填空题；
  - 第三轮：对文中每个高亮词出判断题（释义与用法是否对应）。

### 7.4 剧情/脚本卡

- 进一步玩法（可以做成高级功能）：
  - 把生成的小古文拆成“场景 + 对话气泡”，每一条对话带上一个目标词；
  - 用户可以点击角色台词逐句练习，也可以录音朗读（以后再加）。

## 八、30 日连续打卡计划（“套餐”）

> 图上绿色：✅「打卡套餐：30 日连续计划；转化埋点，结束在 40 日内，优惠/限时/阶梯」

### 8.1 计划创建

- 为每个年级设计固定的“30 日实词表”，例如：
  - 每天 10 个词（6 个复习 + 4 个新词）。
- 数据结构：

```ts
studyPlan = {
  planId: 'junior3_30days',
  totalDays: 30,
  days: [
    { day: 1, wordIds: [...] },
    { day: 2, wordIds: [...] },
    ...
  ]
}
```

### 8.2 执行逻辑

1. 用户点击“开启 30 日计划”：
   - 将 `planId` + `startDate` 写进本地或云端。
2. 每天进入应用时：
   - 计算 `dayIndex = min( today - startDate + 1, 40 )`。
   - 如果 `dayIndex > 30 && 当天任务未完成` → 标记为“计划过期”（用于埋点）。
   - 如果 `1 <= dayIndex <= 30`：
     - 使用 `studyPlan.days[dayIndex]` 作为今日任务的 `wordIds`。
3. 完成当天所有题目：
   - 标记 `finishedDays[dayIndex] = true`。
   - 更新“连续打卡天数”。

> 这块跟前面的“每日打卡”共用同一套练习页，区别只是 wordIds 的来源不同。
