# 数据库设计文档

## 一、云数据库集合（Cloud Database Collections）

### 1. users（用户表）
存储所有用户的基本信息和角色信息。

```javascript
{
  _id: "auto_generated_id",           // 云数据库自动生成
  _openid: "微信openid",               // 云开发自动注入，用户唯一标识

  // 基本信息
  userInfo: {
    nickName: "张三",                 // 微信昵称
    avatarUrl: "https://...",         // 微信头像
  },

  // 角色信息
  role: "student",                    // "student" | "teacher"

  // 学生专属字段（仅 role=student 时有效）
  studentInfo: {
    grade: "七上",                    // 年级
    school: "XX中学",                 // 学校（可选）
    realName: "",                     // 真实姓名（可选）
    studentId: ""                     // 学号（可选）
  },

  // 教师专属字段（仅 role=teacher 时有效）
  teacherInfo: {
    school: "XX中学",                 // 学校
    subject: "语文",                  // 科目
    realName: "李老师",               // 真实姓名
    phone: "138****1234",             // 手机号（可选）
    certificationStatus: "approved",  // "pending" | "approved" | "rejected"
    certificationImages: [],          // 教师资格证图片URL（可选）
    certifiedAt: "2025-11-26T10:00:00.000Z"  // 认证通过时间
  },

  // 会员信息（模块5）
  vipInfo: {
    isVip: false,
    vipType: "free",                  // "free" | "student_monthly" | "teacher_yearly"
    expireAt: null,
    purchaseHistory: []               // 购买记录
  },

  // 统计信息
  stats: {
    totalPracticeCount: 0,            // 总练习次数
    totalCheckInDays: 0,              // 总打卡天数
    checkInStreak: 0,                 // 连续打卡天数
    joinedClassCount: 0,              // 加入的班级数（学生）
    createdClassCount: 0              // 创建的班级数（教师）
  },

  // 时间戳
  createdAt: "2025-11-26T10:00:00.000Z",
  lastLoginAt: "2025-11-26T10:00:00.000Z",
  updatedAt: "2025-11-26T10:00:00.000Z"
}
```

**索引：**
- `_openid` (唯一索引，云开发自带)
- `role` (普通索引)
- `teacherInfo.certificationStatus` (普通索引)

      nextReviewAt: "2025-11-27T10:00:00.000Z",
      masteryLevel: 2                 // 掌握度等级 0-5
    },
    "2": { ... }
  },

  // 每日任务记录（最近30天）
  dailyTasks: {
    "2025-11-26": {
      wordIds: [1, 5, 12, 23, 45, 67, 89, 101, 120, 135],
      oldWordCount: 6,
      newWordCount: 4,
      totalCount: 10,
      finishedCount: 7,

      createdAt: "2025-11-26T08:00:00.000Z"

**索引：**

### 3. classes（班级表）
存储教师创建的班级信息。

```javascript
{
  _id: "auto_generated_id",
  _openid: "教师的openid",            // 班级创建者

  // 班级基本信息
  className: "初二3班",
  grade: "八上",                      // 年级
  school: "XX中学",
  subject: "语文",

  // 邀请码
  joinCode: "ABC123",                 // 6位随机码，唯一
  joinCodeExpireAt: null,             // 邀请码过期时间（null表示永不过期）

  // 班级设置
  settings: {
    allowStudentInvite: false,        // 是否允许学生邀请其他学生
    autoApprove: true,                // 是否自动通过学生加入申请
    maxStudents: 100                  // 最大学生数
  },

  // 统计信息
  stats: {
    studentCount: 0,                  // 学生数量
    pkSessionCount: 0,                // 已创建的PK任务数
    totalPracticeCount: 0             // 班级总练习次数
  },

  // 状态
  status: "active",                   // "active" | "archived"

  // 时间戳
  createdAt: "2025-11-26T10:00:00.000Z",
  updatedAt: "2025-11-26T10:00:00.000Z"
}
```

**索引：**
- `joinCode` (唯一索引)
- `_openid` (普通索引)
- `status` (普通索引)

### 4. class_members（班级成员表）
存储学生加入班级的记录。

```javascript
{
  _id: "auto_generated_id",

  classId: "班级ID",
  studentOpenid: "学生的openid",

  // 加入信息
  joinedAt: "2025-11-26T10:00:00.000Z",
  joinMethod: "code",                 // "code" | "invite" | "teacher_add"

  // 学生信息快照（方便查询）
  studentInfo: {
    nickName: "张三",
    avatarUrl: "https://...",
    grade: "八上",
    realName: "",                     // 如果学生填写了真实姓名
    studentId: ""                     // 如果学生填写了学号
  },

  // 班级内统计
  classStats: {
    totalPkCount: 0,                  // 参与的PK次数
    totalScore: 0,                    // 总得分
    averageScore: 0,                  // 平均分
    ranking: 0                        // 班级排名
  },

  // 状态
  status: "active",                   // "active" | "left" | "removed"

  updatedAt: "2025-11-26T10:00:00.000Z"
}
```

**索引：**
- `classId` + `studentOpenid` (复合唯一索引)
- `classId` (普通索引)
- `studentOpenid` (普通索引)

### 5. pk_sessions（PK任务表）
存储教师创建的PK任务。

```javascript
{
  _id: "auto_generated_id",
  _openid: "教师的openid",

  classId: "班级ID",

  // 任务基本信息
  title: "第一单元实词测验",
  description: "涵盖第一单元所有实词",

  // 题目设置
  wordIds: [1, 5, 12, 23, 45],        // 选择的词ID列表
  questionCount: 20,                  // 生成的题目总数
  timeLimit: 600,                     // 时限（秒），0表示不限时

  // 难度设置
  difficulty: "mixed",                // "easy" | "medium" | "hard" | "mixed"

  // PK状态
  status: "pending",                  // "pending" | "running" | "finished"
  startTime: "2025-11-26T14:00:00.000Z",  // 开始时间
  endTime: "2025-11-26T15:00:00.000Z",    // 结束时间

  // 统计信息
  stats: {
    participantCount: 0,              // 参与人数
    completedCount: 0,                // 完成人数
    averageScore: 0,                  // 平均分
    averageTime: 0                    // 平均用时（秒）
  },

  // 时间戳
  createdAt: "2025-11-26T10:00:00.000Z",
  updatedAt: "2025-11-26T10:00:00.000Z"
}
```

**索引：**
- `classId` (普通索引)
- `status` (普通索引)
- `startTime` (普通索引)

### 6. pk_records（PK记录表）
存储学生参与PK的答题记录。

```javascript
{
  _id: "auto_generated_id",
  _openid: "学生的openid",

  pkSessionId: "PK任务ID",
  classId: "班级ID",

  // 答题结果
  score: 85,                          // 得分
  totalQuestions: 20,                 // 总题数
  correctCount: 17,                   // 答对数
  wrongCount: 3,                      // 答错数

  // 用时
  startTime: "2025-11-26T14:05:00.000Z",
  endTime: "2025-11-26T14:15:30.000Z",
  duration: 630,                      // 用时（秒）

  // 详细答题记录
  answers: [
    {
      questionId: 1,
      wordId: 5,
      isCorrect: true,
      selectedOption: "A",
      correctOption: "A",
      timeCost: 10                    // 该题用时（秒）
    },
    // ...
  ],

  // 错题统计
  wrongWords: [12, 23, 45],           // 答错的词ID列表

  // 排名（在该PK中）
  ranking: 5,

  // 时间戳
  submittedAt: "2025-11-26T14:15:30.000Z"
}
```

**索引：**
- `pkSessionId` + `_openid` (复合唯一索引)
- `pkSessionId` (普通索引)
- `_openid` (普通索引)

## 三、数据迁移策略

### 从本地到云端的迁移：
1. 用户首次云端登录时，检测本地是否有学习进度数据
2. 如果有，提示用户"检测到本地学习记录，是否上传到云端？"
3. 用户同意后，调用云函数 `syncLocalProgress` 上传数据
4. 上传成功后，本地数据标记为"已同步"

### 云端到本地的同步：
1. 用户登录后，自动拉取云端进度数据
2. 对比本地和云端的 `lastSyncAt` 时间戳
3. 如果云端更新，下载并覆盖本地数据
4. 如果本地更新，上传到云端

## 四、权限设计

### 云数据库权限规则：

**users 表：**
- 读：仅本人可读（`doc._openid == auth.openid`）
- 写：仅本人可写
- 创建：任何登录用户

**user_progress 表：**
- 读：仅本人可读
- 写：仅本人可写
- 创建：任何登录用户

**classes 表：**
- 读：
  - 班级创建者（教师）
  - 班级成员（学生）
- 写：仅班级创建者
- 创建：仅教师角色

**class_members 表：**
- 读：
  - 班级创建者
  - 本人的记录
- 写：
  - 班级创建者（可以移除学生）
  - 学生可以创建加入记录
- 删除：仅班级创建者

**pk_sessions 表：**
- 读：班级创建者和班级成员
- 写：仅班级创建者
- 创建：仅班级创建者

**pk_records 表：**
- 读：
  - 记录创建者（学生本人）
  - 班级创建者（教师）
- 写：仅记录创建者
- 创建：班级成员

## 五、云函数设计

### 1. login（登录）
- 输入：微信 code
- 输出：openid, session_key, 用户信息
- 逻辑：
  1. 调用微信 API 换取 openid
  2. 查询 users 表是否存在该用户
  3. 如果不存在，创建新用户记录（role 默认为空，需要后续选择）
  4. 更新 lastLoginAt
  5. 返回用户信息

### 2. updateUserRole（设置/更新用户角色）
- 输入：role, roleInfo（学生信息或教师信息）
- 输出：更新后的用户信息
- 逻辑：
  1. 验证 role 有效性
  2. 如果是教师，设置 certificationStatus 为 "pending"
  3. 更新 users 表
  4. 返回新的用户信息

### 3. syncProgress（同步学习进度）
- 输入：本地学习进度数据
- 输出：云端最新进度数据
- 逻辑：
  1. 对比本地和云端的 lastSyncAt
  2. 合并数据（以最新的为准）
  3. 更新 user_progress 表
  4. 返回最新数据

### 4. createClass（创建班级）
- 输入：班级信息
- 输出：班级ID 和 邀请码
- 逻辑：
  1. 验证用户是教师身份
  2. 生成唯一邀请码
  3. 创建 classes 记录
  4. 返回班级信息

### 5. joinClass（加入班级）
- 输入：邀请码
- 输出：班级信息
- 逻辑：
  1. 验证邀请码有效性
  2. 验证用户是学生身份
  3. 检查是否已加入该班级
  4. 创建 class_members 记录
  5. 更新 classes 的 studentCount
  6. 返回班级信息

### 6. createPkSession（创建PK任务）
- 输入：PK任务配置
- 输出：PK任务ID
- 逻辑：
  1. 验证用户是该班级的教师
  2. 生成题目（根据 wordIds）
  3. 创建 pk_sessions 记录
  4. 返回任务信息

### 7. submitPkRecord（提交PK答题记录）
- 输入：PK任务ID, 答题记录
- 输出：得分、排名
- 逻辑：
  1. 验证用户是班级成员
  2. 计算得分和用时
  3. 创建 pk_records 记录
  4. 更新 pk_sessions 的统计信息
  5. 计算排名
  6. 返回结果

## 六、数据安全

### 1. 敏感信息加密
- 手机号：脱敏显示（138****1234）
- 真实姓名：仅教师和本人可见

### 2. 数据备份
- 定期备份云数据库
- 重要操作记录日志

### 3. 防刷机制
- PK提交时验证答题时间合理性
- 防止短时间内重复提交

## 七、数据量估算

假设：
- 总用户数：10,000
- 教师比例：5%（500人）
- 平均每个教师创建班级：2个
- 平均每个班级学生：30人
- 平均每个班级PK任务：10个/学期

**数据量：**
- users: 10,000 条
- user_progress: 10,000 条
- classes: 1,000 条
- class_members: 30,000 条
- pk_sessions: 10,000 条
- pk_records: 300,000 条

**存储空间估算：**
- 总计约 100MB - 500MB（取决于详细答题记录的存储）

**微信云开发免费额度：**
- 数据库：2GB
- 云函数调用：10万次/月
- 云存储：5GB

**结论：**免费额度足够支撑初期使用。
